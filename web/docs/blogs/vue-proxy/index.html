    defineProperty与proxy对象绑定

    本文是比较Object.defineProperty() 与 proxy代理对于对象双向数据绑定原理以及罗列对应的优缺点

    # Object.defineProperty() 与 proxy代理优缺点

    Object.defineProperty() 直接在一个对象上定义一个新属性，或者修改一个对象的现有属性，并返回此对象

    缺点: 1、无法监听数组的变化(比如push, pop, shift, unshift,splice, sort, reverse); 2、只能监听属性，而不是监听对象本身，需要对对象的每个属性进行遍历。对于原本不在对象中的属性难以监听。

    proxy代理 基于对象创建一个对象的代理，实现基本操作的拦截和自定义（如属性查找、赋值、枚举、函数调用）

    优点: 1、可以监听数组变化; 2、监听的是对象本身，可以获取到对象里面具体哪些属性做了修改，扩大了对 对象操作的空间; 3、向外提供了有13种拦截方法，更大程度的给开发者提供了强大的api功能 13种拦截方式文档地址 (opens new window)

    # proxy监听对象代码样例

    <template>
    <div class="proxy">
        <h1>proxy  defineProperty  test</h1>
        <br>
        <label for="">名字：</label>
        <input type="text" v-model="obj.name">
        <br>
        <label for="">年龄：</label>
        <input type="text" v-model="obj.age">
    </div>
</template>

<script>
    export default {
        name: 'proxy',
        data() {
            return {
                obj: {}
            }
        },
        mounted() {
            this.test();
        },
        methods: {
            test() {
               this.obj = new Proxy({name: '张三', age: 26}, {
                   set: function(obj, prop, value) {
                       console.log(obj, prop, value);
                       if (obj[prop] !== value) console.log(`更改了的key是${prop}`);
                       // 表示成功
                       return true;
                   }
               })
               Object.defineProperty(this.obj, 'name', {
                   get: function(a, b, c) {
                       console.log(a, b, c);
                       return null;
                   },
                   set: function(a, b, c) {
                        console.log(a, b, c);
                   }
               });
            }
        }
    }
</script>
